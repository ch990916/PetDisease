[유스케이스 시나리오]
	
	회원 탈퇴
	
	
목표 : 사용자는 가입한 정보에 접근하여 회원을 탈퇴할 수 있다.


사전조건 : 로그인이 되어있어야 한다.


성공시나리오

	1. 사용자가 회원 정보 버튼을 클릭한다.
	
	2. 시스템은 기존 가입 정보를 불러와 회원 정보 페이지를 출력한다.
	
	3. 사용자가 회원 탈퇴 버튼을 클릭한다.
	
	4. 시스템은 ID, PW를 한번 더 입력받을 수 있게 화면을 출력한다(확인/취소).
	
	5. 사용자는 ID, PW 입력후 확인 버튼을 클릭한다.
	
	6. 시스템은 탈퇴 확인 여부를 묻는 확인/취소 버튼을 출력한다.
	
	7. 사용자는 확인 버튼을 클릭한다.
	
	8. 시스템은 사용자 로그아웃을 진행하고 사용자 정보를 삭제한다.
	
	9. 시스템은 탈퇴 완료 알람을 띄우고 메인페이지를 출력한다.
		
		
		
확장시나리오
	
	2.1. 회원 정보 페이지 관련 :
		2.1.1. 회원정보 페이지에서 수정 및 탈퇴를 진행할 수 있으며
			  취소버튼을 통해 페이지를 벗어날 수 있다.
		
	4.1. ID, PW가 틀리는 경우 :
		4.1.1. 시스템은 입력한 정보가 올바르지 않다는 알람을 출력한다.
	
	4.2. ID, PW가 3회 이상 틀리는 경우 :
		4.2.1. 시스템은 사용자를 로그아웃 하고 메인페이지를 출력한다.
	
	4.3. ID, PW 확인 화면에서 취소버튼을 클릭하는 경우 :
		4.3.1. 회원 정보 페이지를 재 출력한다.
		
	7.1. 취소 버튼을 클릭하는 경우 :
		7.1.1. 회원 정보 페이지를 재 출력한다.
